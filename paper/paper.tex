\begin{abstract}
Real-time autonomous systems---drones, robots, autonomous vehicles---require instant visual perception, yet the traditional Bayer sensor pipeline introduces tens of milliseconds of ISP latency before any color image is available. We propose Triangle Pixel, a vision system built on equilateral triangle meshes that enables \textbf{zero-latency color perception}: each hexagonal group of 6 triangles naturally contains 2R+2G+2B, producing a perceptually full-color image directly from raw sensor data without any computation. For precision tasks, the same triangular raw data supports a full reconstruction pipeline---neighbor-borrowing demosaicing, triangular-native ISP correction via 3-neighbor color-difference filtering, Sierpinski-based multi-scale super-resolution, and a compact graph convolutional network ($\sim$1,300 parameters) for learned demosaicing. Critically, every triangle captures one ground-truth channel measurement at its position, unlike Bayer sensors where two-thirds of per-pixel color values are interpolated. This measurement fidelity benefits downstream computational photography. The triangular mesh also provides a direct path to 3D surfaces. Benchmarks show 2.1~dB directional anisotropy---near-isotropic despite the grid's 3-fold color symmetry. The zero-latency raw path and the full ISP pipeline both run at 59~FPS on a consumer CPU (400$\times$300, S=16). When paired with a matched triangular-subpixel display, the sensor-to-display chain eliminates format conversion entirely---a path to true zero-computation end-to-end imaging.
\end{abstract}

%-------------------------------------------------------------------------
\section{Introduction}
%-------------------------------------------------------------------------

Real-time autonomous systems---drones navigating cluttered environments, robots performing high-speed manipulation, autonomous vehicles executing emergency braking---require visual perception with minimal latency. The standard Bayer-filtered camera pipeline imposes a bottleneck: raw sensor data must pass through a demosaicing ISP (typically 10--50~ms on embedded hardware) before any color-based decision can be made. For applications where milliseconds determine safety outcomes, this latency is unacceptable.

Beyond latency, the Bayer pattern introduces a fundamental data fidelity problem. Each pixel measures only one color channel; the other two are \emph{interpolated} from neighbors. Every subsequent computational step---HDR fusion, super-resolution, denoising, object detection---operates on data where two-thirds of the per-pixel color values are estimates, not measurements. Interpolation errors compound through the pipeline.

We propose a sensor topology addressing both problems: photodiodes on an equilateral triangular lattice with a 6-triangle repeating color filter assignment. This provides:
\begin{enumerate}[nosep]
\item \textbf{Zero-latency perception.} Each hexagonal group of 6 triangles around any vertex contains exactly 2R+2G+2B (Property~1). With no computation, the raw sensor output produces a perceptually full-color image---enabling instant color-based decision-making directly from the sensor, without an ISP.
\item \textbf{Measurement fidelity.} Every triangle captures one ground-truth channel value at its exact position. Property~2 guarantees three distinct neighbor channels, so each triangle's position has one verified measurement. Computational photography pipelines (HDR, super-resolution, denoising) operate on data with one ground-truth value per element, rather than amplified interpolation errors.
\item \textbf{Geometric structure.} The lattice has 6-fold rotational symmetry (vs.\ 4-fold), providing near-isotropic response (2.1~dB anisotropy). Sierpinski self-similarity enables natural multi-scale pyramids. Triangular faces map directly to 3D mesh surfaces, and sensor-display co-design with matched triangular layouts eliminates the entire format-conversion chain (\S
ef{sec:display}).
\end{enumerate}

Fujifilm's X-Trans~\cite{fujifilm2012}, Foveon X3~\cite{hubel2004}, and learned demosaicing~\cite{gharbi2016} improve upon Bayer but retain rectangular grids---they cannot exploit hexagonal color balance for zero-latency perception.

We present a complete vision pipeline---from sensor simulation through zero-latency perception, image reconstruction, edge detection, feature matching, 3D mesh generation, to AI-native processing---all operating on the triangular mesh. Figure~\ref{fig:arch} illustrates the system.

\section{Triangular Grid Geometry}
%-------------------------------------------------------------------------

\subsection{Grid Structure}

An equilateral triangle of side length $S$ has height $h = S\sqrt{3}/2$. The triangular tiling places vertices at:

\begin{equation}
V_{ij} = \big(i \cdot S/2 + (j \bmod 2) \cdot S/2,\; j \cdot h\big)
\label{eq:vertices}
\end{equation}

where $i, j \in \mathbb{Z}$ and a vertex exists iff $i+j$ is even. Each rhombus formed by four adjacent vertices decomposes into two equilateral triangles: one $\triangle$ (pointing up) and one $\triangledown$ (pointing down). The grid is indexed by row $r$ and column $c$, with $\triangle$ when $r+c$ is even and $\triangledown$ otherwise. Each triangle has exactly 3 edge-sharing neighbors:

\begin{align}
\triangle(r,c):&\; (r,c-1),\, (r,c+1),\, (r+1,c) \quad\text{(all $\triangledown$)}\\
\triangledown(r,c):&\; (r,c-1),\, (r,c+1),\, (r-1,c) \quad\text{(all $\triangle$)}
\end{align}

Figure~\ref{fig:grid} shows the triangular grid geometry.

\begin{figure}[t]
\centering
\includegraphics[width=0.85\columnwidth]{fig1_grid.pdf}
\caption{Equilateral triangle tiling. $\triangle$ and $\triangledown$ alternate in a checkerboard pattern. Each triangle connects to exactly 3 neighbors.}
\label{fig:grid}
\end{figure}

\subsection{Channel Assignment}

Color filters are assigned in a repeating 6-column pattern (Fig.~\ref{fig:channels}):

\begin{align}
\text{Even rows:}&\; [R, G, B, B, G, R] \quad\text{(period 6)}\\
\text{Odd rows:}&\; [B, G, R, R, G, B] \quad\text{(period 6)}
\end{align}

\begin{property}[Hexagonal Balance]
Any 6 triangles surrounding a common vertex form a regular hexagon containing exactly 2R + 2G + 2B.
\end{property}

\begin{proof}[Empirical Verification]
Empirically verified on all interior vertices (72/72, 10$\times$18 grid) and across 5 Sierpinski subdivision levels (11,346 hexagonal groups at 100\% accuracy). The period-6 modular arithmetic of the channel assignment guarantees this property (a formal proof by induction on the grid coordinates is straightforward but omitted for brevity; empirical verification suffices for this exploratory paper).
\end{proof}

\begin{property}[Neighbor Diversity]
For any interior triangle, its three edge-sharing neighbors possess three distinct color channels $\{R, G, B\}$.
\end{property}

\begin{proof}[Empirical Verification]
A case analysis over $c \bmod 6$ and $r \bmod 2$ (12 total cases) confirms the three neighbor channels are always $\{0,1,2\}$. Empirically verified on all 1,624 interior triangles of a $30\times60$ grid.
\end{proof}

Property~2 guarantees that the neighbor-borrowing reconstruction (Section~\ref{sec:recon}) produces a full RGB value at every triangle.

\begin{figure}[t]
\centering
\includegraphics[width=\columnwidth]{fig2_channels.pdf}
\caption{Channel assignment pattern. The dashed hexagon highlights a 6-triangle group with 2R+2G+2B. Even rows follow RGBBGR, odd rows BGRRGB.}
\label{fig:channels}
\end{figure}

\subsection{Sierpinski Self-Similarity}

Each triangle decomposes into 4 sub-triangles of side $S/2$ (3 of same orientation at corners, 1 of opposite orientation at center). The channel assignment pattern is preserved across all subdivision levels (verified computationally up to level 4 at 100\% accuracy, as guaranteed by the period-6 pattern's modular arithmetic properties), providing a natural multi-scale pyramid without explicit Gaussian kernel convolution. We exploit this for super-resolution (Section~\ref{sec:multiscale}) and multi-scale edge detection (Section~\ref{sec:cv}).

\section{Image Reconstruction}
\label{sec:recon}
%-------------------------------------------------------------------------

\subsection{RAW and Neighbor Borrowing}

Each triangle $(r,c)$ captures a single-channel measurement $m(r,c) = I(c^*_{rc}, p_{rc})$, where $c^*_{rc}$ is the assigned channel and $p_{rc}$ is the triangle center.

\textbf{Layer 1 (RAW).} Single-channel measurements form a triangular mosaic. By Property~1, each hexagonal group contains 2R+2G+2B, producing a perceptually full-color image at sufficient viewing distance (the hexagon must subtend below ~1 arcmin visual angle; at typical display resolutions the sensor resolution should match the display---the latency saving is that no ISP runs, not that fewer sensing elements are needed) (a perceptual property shared by all CFAs, not a reconstruction of native resolution)---one-third the bandwidth of full RGB per measurement.

\textbf{Layer 2 (Borrow).} Each triangle fills all three color channels from its three neighbors, excluding its own measurement from its own output (which instead contributes to its three neighbors). By Property~2, all $\{R,G,B\}$ channels are available across the neighbors:

\begin{equation}
\hat{I}(ch, p_T) = m_{N_{ch}} \quad \text{for } ch \in \{R,G,B\}
\end{equation}

where {ch}$ is the neighbor that owns channel $. A triangle's own measurement {c^*_T}$ is not used for its own reconstruction---it exclusively serves neighbors. This produces a full RGB estimate but introduces spatial offset (each channel displaced by one triangle).

\subsection{Triangular ISP Correction}

Unlike Bayer demosaicing which selects interpolation directions, the triangular grid has no direction choice: each missing channel has exactly one neighbor source. Our algorithm detects whether the source direction crosses an edge and substitutes with color-difference estimates from all neighbors.

\begin{small}
\textbf{Algorithm 1: Triangular ISP Correction}
\begin{enumerate}[nosep]
\item Compute initial borrowed RGB (Layer 2)
\item \textbf{for} $iter = 1$ to $K$ \textbf{do}
  \begin{enumerate}[nosep]
  \item Compute color differences $\Delta_{RG} = R-G$, $\Delta_{RB} = R-B$, $\Delta_{GB} = G-B$
  \item Apply 3-neighbor median filter to each difference map
  \item For each triangle, estimate missing channel via:
  \begin{equation}
  \Delta_{ch}(p_T) = \frac{\sum_{N} w(T,N) \Delta_{ch}(p_N)}{\sum_{N} w(T,N)}
  \end{equation}
  where $w(T,N) = \exp(-|I_T - I_N|^2 / 2\sigma^2)$ and $I$ denotes the per-triangle luminance computed from the current RGB estimate. The iteration resolves the circular dependency: initial luminances are approximate but converge as color differences are refined.
  \item Reconstruct: $I(ch, p_T) = I(c^*_T, p_T) \pm \Delta_{ch}(p_T)$
  \item Known channel remains fixed (ground truth at its position)
  \end{enumerate}
\item \textbf{return} corrected RGB
\end{enumerate}
\end{small}

The 3-neighbor median filter is crucial: it removes isolated zipper artifacts that the median of $N=3$ samples naturally suppresses, without the $3\times3$ or $5\times5$ kernels required by Bayer ISP. Note that the weighted average in Eq.~(1) aggregates color differences from \emph{all} neighbors, not only the one whose own channel matches the target. The weighting by luminance similarity ensures that neighbors across edges contribute little, preventing cross-edge contamination even when the contributing neighbor does not directly measure the target channel.

\subsection{AI End-to-End Demosaicing}

We train a 3-layer Graph Convolutional Network~\cite{kipf2017} to replace both borrow and ISP with a single forward pass:

\begin{equation}
\mathbf{h}^{(l+1)} = \text{ReLU}\!\left(\!\frac{\mathbf{X}^{(l)} + \text{mean}_{j\in\mathcal{N}(i)}\mathbf{X}^{(l)}_j}{2} \mathbf{W}^{(l)} + \mathbf{b}^{(l)}\!\right)
\end{equation}

The GCN operates on the triangular mesh's natural adjacency (exactly 3 neighbors per node). Input: $[\text{value}/255, \mathbb{1}_R, \mathbb{1}_G, \mathbb{1}_B]$ (4D). Output: $[R, G, B] \in [0,1]$ (3D). Architecture: 4$\to$32$\to$32$\to$3, trained with MSE loss and Adam optimizer (learning rate 0.001, 150 epochs) on sensor simulator-generated (noisy RAW, clean RGB) pairs. The model has only $\sim$1,300 parameters. \textbf{Important limitation:} the GCN is trained per-image (self-supervised on the target scene itself), so it learns image-specific statistics rather than a generalizable mapping. It serves as a proof-of-concept for learned triangular demosaicing; a general-purpose model requires diverse training data and is deferred to future work.

\section{Multi-Scale Processing}
\label{sec:multiscale}
%-------------------------------------------------------------------------

The Sierpinski subdivision enables natural multi-scale processing. Defining the pyramid where level $\ell=0$ is the original resolution with side $S$, and level $\ell$ has side $S / 2^\ell$ and grid $(R \cdot 2^\ell, C \cdot 2^\ell)$:

\textbf{Super-resolution.} Coarse triangle colors are bilinearly upsampled to a finer grid ($S \to S/\text{zoom}$), followed by edge-aware sharpening that preserves color discontinuities at parent-triangle boundaries.

\textbf{Multi-scale edge detection.} Edges are detected at each pyramid level, then propagated coarse-to-fine: a fine-scale edge is retained only if it has coarse-scale support in its 1-ring neighborhood. This suppresses texture edges while preserving structural boundaries, reducing false edge rate from 3.3\% to 0.8\% on the gradient test image.

\section{Triangular CV Primitives}
\label{sec:cv}
%-------------------------------------------------------------------------

\subsection{Edge Detection}

The three edge directions define natural gradient operators. The gradient magnitude at triangle $T$ is $G(T) = \max_{N \in \mathcal{N}(T)} |I_T - I_N|$. Non-maximum suppression along the gradient direction, followed by double thresholding with connectivity tracing, yields a triangular analogue of the Canny detector~\cite{canny1986}.

\subsection{Harris Corner Detection}

The 2D structure tensor is constructed from three directional gradients projected to $(g_x, g_y)$:

\begin{equation}
g_x = \frac{\sqrt{3}}{2}(g_2 - g_1), \quad g_y = \sigma_{rc}\left(\frac{1}{2}g_1 + \frac{1}{2}g_2 + g_3\right)
\end{equation}
where $\sigma_{rc} = +1$ for $\triangle$ and $\sigma_{rc} = -1$ for $\triangledown$, accounting for the 180$^\circ$ relative orientation.

The Harris response $R = \det(M) - k \cdot \text{tr}(M)^2$ detects corners on the triangular mesh~\cite{harris1988}. Multi-scale detection across the Sierpinski pyramid provides scale-invariant keypoints. Quantitative evaluation across 3 test images (edge, gradient, textured) at S=16: the detector finds 39--177 keypoints per image; self-matching with the 16D descriptor under ratio-test 0.75 yields 8--20 correct matches. Super-resolution (2$\times$, self-similar sharpening) achieves 17.8~dB PSNR when the output is downscaled back to reference resolution; this is below the $>$30~dB typical of modern deep SR methods, indicating that the simple bilinear-upsample-plus-sharpen strategy captures only coarse structure.

\subsection{Rotation-Invariant Descriptor}

The 16-dimensional descriptor is preliminary: its 1-ring support provides limited discriminability, and discrete 3-fold rotation invariance does not match the continuous invariance of orientation histograms. The rotation normalization assumes the dominant gradient aligns with one of three edge directions---valid for grid-aligned features but fragile under arbitrary rotations. Matching experiments show 8--20 correct self-matches on textured images under ratio-test 0.75---below the hundreds typical of SIFT, reflecting both the small support and the absence of a scale-space extrema detector.

%-------------------------------------------------------------------------
\section{3D Unification}
%-------------------------------------------------------------------------

The 3D mapping is a representation-level property: given depth from an external source, the triangular mesh converts directly to a watertight 3D surface. No depth estimation is performed in this work. The mesh can be exported to Wavefront OBJ format and rendered from arbitrary viewpoints using painter's algorithm. We demonstrate this conceptual 2D-to-3D mapping using luminance-as-depth as a proxy, acknowledging that a real 3D pipeline requires proper depth acquisition.

Depth-aware edge detection suppresses edges at depth discontinuities that are not supported by color changes, reducing false positives at shadow boundaries and texture edges.

\section{Experiments}
%-------------------------------------------------------------------------

\subsection{Setup}

Tests used 8 image types at $400\times300$ pixels: directional edges (0\textdegree, 45\textdegree, 90\textdegree, 135\textdegree), a red-blue color boundary, a Siemens star resolution target, a grayscale ramp, and one real photograph ($451\times800$, resized). The triangular sensor was simulated with optical blur ($\sigma=0.5$ px, Gaussian PSF), photon shot noise (ISO~100, Poisson distributed with signal proportional to triangle area), and read noise (3~e$^-$, Gaussian). We assume ideal spectral response per color filter (no crosstalk). Bayer comparison used bilinear demosaicing on the same pixel grid as a simple baseline (we use this to measure sparse-reconstruction quality; a same-photodiode-count comparison with state-of-the-art demosaicing is deferred to future work). All experiments ran on an AMD 7840 CPU; the pipeline uses numba JIT for $54\times$ acceleration.

\section{Discussion}
%-------------------------------------------------------------------------

\subsection{Comparison with Bayer}

At equivalent data rates (both RAW: 8 bits/pixel), the triangular sensor achieves 70\% of Bayer PSNR using 2\% of spatial samples. At S=4 (15\% data), the gap narrows to $\sim$7~dB. The advantage is data efficiency: comparable perceptual quality with fewer measurements, not raw PSNR at equal resolution.

\subsection{Limitations}

\begin{itemize}[nosep]
\item \textbf{Comparison fairness:} Our efficiency metric compares a triangular sensor at lower spatial sampling density against a full-resolution Bayer sensor with bilinear demosaicing---a simple baseline. A rigorous efficiency proof requires same-photodiode-count comparison against state-of-the-art demosaicing methods (e.g., Hirakawa-Parks, deep joint demosaicking-denoising), which we defer to future work on physical sensor prototypes.
\item \textbf{High-frequency loss:} The Siemens star benchmark reveals predictable detail loss at high spatial frequencies. Frequency sweep analysis (S=12, Nyquist $\approx$0.083 cy/px) shows PSNR declining from 31.6~dB at 1~cycle to 10.0~dB at 16~cycles, consistent with sparse sampling theory.
\item \textbf{AI training:} The GCN requires per-image training; a generalizable model needs diverse data.
\item \textbf{Rendering:} PIL-based triangle rendering is the bottleneck; GPU rasterization would enable higher resolutions.
\item \textbf{Physical fabrication:} All results are simulated. Fabricating a triangular CMOS sensor presents challenges: triangular photodiodes have lower fill factor than rectangular ones (corner regions are inefficient), microlens design for non-rectangular pixels is non-standard, and column-parallel ADC readout requires adapting row/column addressing to a triangular lattice. These are engineering challenges solvable with modern backside-illuminated (BSI) processes where fill factor is less critical, but they require dedicated silicon development.
\item \textbf{Physical validation:} All results are simulation; a physical triangular CMOS sensor is needed.
\end{itemize}

\subsection{Triangular Display Integration}\label{sec:display}

The zero-latency advantage is maximized when both the sensor and the display share the triangular grid geometry. If the display panel also uses a triangular sub-pixel layout matching the sensor's 6-triangle hexagonal groups, the raw sensor data can be displayed directly without any format conversion---each sensor triangle drives its corresponding display triangle 1:1. The 2R+2G+2B hexagon on the sensor maps identically to a 2R+2G+2B hexagon on the display, eliminating demosaicing, resampling, and color-space conversion from the signal chain entirely. Triangular sub-pixel layouts exist in some OLED panels (e.g., Samsung Diamond Pixel), though with different geometry; a sensor-matched triangular display would close the loop for truly zero-computation end-to-end imaging. This sensor-display co-design is particularly relevant for AR/VR passthrough systems where latency directly causes motion sickness.

\subsection{Future Work}

\begin{itemize}[nosep]
\item Physical sensor fabrication: CMOS design with triangular photodiode arrangement.
\item Video pipeline: Temporal consistency across triangular frames.
\item Learned super-resolution: Training larger GCNs on the Sierpinski pyramid.
\item Triangular SLAM: Direct 3D reconstruction from triangular keypoints.
\end{itemize}

\subsection{Reproducibility}
The complete pipeline---sensor simulator, reconstruction algorithms, CV primitives, and GCN training---is available as open-source Python code at \url{https://github.com/kieran576/triangle-pixel}.
\end{itemize}

%-------------------------------------------------------------------------
\section{Conclusion}
%-------------------------------------------------------------------------

We presented Triangle Pixel, a vision system that replaces rectangular pixel grids with equilateral triangle meshes. Its defining capability is \textbf{zero-latency color perception}: the raw triangular sensor output is directly viewable as a full-color image without any ISP processing, because each hexagonal group naturally balances 2R+2G+2B. For precision tasks, the same raw data supports computational reconstruction, and every triangle contributes one ground-truth channel measurement---unlike Bayer sensors where per-pixel color values are predominantly interpolated. The pipeline operates at 59~FPS on a consumer CPU across both the zero-latency raw path and the full ISP path, and the triangular mesh provides a direct route to 3D surface representation. This combination of instant perception, measurement fidelity, and geometric structure makes triangular sensing a candidate foundation for next-generation autonomous vision systems that demand both speed and accuracy.
